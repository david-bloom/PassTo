## Design Decision - 2026-06-04: Stripe Integration Pattern

**Approved Approach:** Redirect-based Checkout (not embedded)

**Lovable Integration Pattern:**
```javascript
// /payment route: "Subscribe & Continue" handler
const { checkout_url } = await fetch('/.../stripe-checkout-create', {
  method: 'POST',
  headers: { Authorization: `Bearer ${jwt}` }
}).then(r => r.json());

window.location.href = checkout_url; // → Stripe Checkout
// User returns to: enroll.passtodigital.com/upload-selfie (success_url)
// Webhook auto-advances: onboarding_step = 'selfie'
```

**Why This Approach:**
- ✅ Backend (`stripe-checkout-create` v7) already deployed and tested
- ✅ Unblocks TASK-0060 for immediate real payment testing
- ✅ Webhook integration proven (payments, subscriptions, stripe_events tables ready)
- ✅ Simplest path to close acceptance criteria today

**Future Enhancement:**
- After TASK-0060 closes, consider Lovable's embedded checkout (iframe, `enable_stripe_payments`)
- Better UX (no redirect), same webhook backend
- Defer to Phase 7+ pending product prioritization

**Lovable Implementation Guidance:**
1. Replace mock payment buttons in `/payment` route

### Critical Blocker: Lovable /payment Route Incomplete

**Evidence:** Real checkout attempt via Lovable shows:
1. User selected "Credit or Debit Card" and clicked "Subscribe & Continue"
2. Lovable redirected directly to `/upload-selfie` without:
   - Calling `stripe-checkout-create`
   - Showing Stripe Checkout form
   - Requesting payment
3. Edge function logs: **Zero calls to `stripe-checkout-create`** in past 48 hours
4. Selfie upload then failed (payment state not confirmed by webhook)

**Root Cause:** The Lovable `/payment` route is missing the `stripe-checkout-create` integration. It should:
```javascript
// Step 1: Call stripe-checkout-create (with JWT)
const { checkout_url } = await fetch('/.../stripe-checkout-create', {
  headers: { Authorization: `Bearer ${jwt}` }
}).then(r => r.json());

// Step 2: Redirect to Stripe Checkout
window.location.href = checkout_url;

// Step 3: After payment, webhook advances onboarding_step to 'selfie'
```

Currently, the route bypasses all payment processing and directly advances to selfie step.

**TASK-0060 Verdict:** BLOCKED

Cannot complete real Stripe checkout (acceptance criteria #1) until Lovable `/payment` route is wired to call `stripe-checkout-create` and redirect to the returned `checkout_url`.
