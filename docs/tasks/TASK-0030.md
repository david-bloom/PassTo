## Product Goal

Give PassTo operations enough Supabase-based visibility to inspect where a nurse, credential, wallet, verifier, or payment journey failed before production launch.

## Technical Scope

- Create or document Supabase dashboard/views/queries for launch-critical journey inspection.
- Ensure account, identity, phone, license, data match, selfie, credential, wallet, share-link, verifier, Stripe, and entitlement failures are visible.
- Ensure critical failures write audit/event records.
- Document recovery owner/path for each launch-critical journey failure.
- Keep Lovable admin UI deferred.

## Out of Scope

- Full Lovable admin UI.
- Employer dashboard.
- Deep analytics.
- Production launch approval.

## Acceptance Criteria

- [ ] Ops can identify current onboarding/journey step for a nurse.
- [ ] Ops can inspect license, credential, wallet, verifier, payment, and failure state.
- [ ] Critical failure states from PRD Section 3 are represented in event/audit state or operational queries.
- [ ] Supabase views use security-safe patterns and do not weaken RLS/client access.
- [ ] Recovery path/owner is documented for each launch-critical failure state.
- [ ] Claude documents tests, files changed, deviations, and risks.

## QA Plan

- Verify ops queries/views against seeded or test journey states.
- Review security posture for any views or functions.
- Confirm Lovable client cannot access privileged ops data.
