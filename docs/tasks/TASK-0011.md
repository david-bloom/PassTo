## Product Goal

The P2 enrollment pipeline (at `enroll.passtodigital.com`) is currently split across Make.com, Airtable, and a partial Supabase Edge Function. Make handles ID.me code exchange and license lookup trigger. Airtable record IDs serve as the primary nurse identifier throughout the entire flow. The ID.me callback is handled in the browser in violation of platform ownership rules. Wallet pass signing is defined in Vercel but not wired to any Lovable project.

This task produces a complete technical specification for migrating the P2 enrollment pipeline to the approved architecture: Supabase Edge Functions for backend orchestration, Supabase Auth UUIDs as the nurse primary key, ID.me code exchange moved server-side, and wallet pass signing wired end-to-end.

This spec is the prerequisite for all P2 implementation tasks. No implementation work should proceed on P2 without this spec approved.
