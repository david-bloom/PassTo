## Product Goal

Create the backend workflow for `/license-info`: collect the nurse's primary license details, run license lookup, normalize source status, and bind the license result to the verified ID.me identity before plan selection or payment.

## Technical Scope

- Implement or update the backend endpoint/Edge Function used by `/license-info`.
- Accept primary license inputs from Lovable:
  - `license_number`
  - `license_state`
  - `license_type`
- Require a verified ID.me-first onboarding attempt/profile before lookup.
- Call the approved license lookup source server-side only.
- Store lookup source result and normalized status fields using canonical naming.
- Run deterministic ID.me/license data matching after lookup succeeds.
- Persist match result and failure state.
- Advance onboarding only when lookup and match pass.
- Block `/account-select` and payment if lookup or match fails.
- Write audit/failure events for lookup success, lookup failure, status failure, and identity/license mismatch.

## Out of Scope

- ID.me-first callback/account-link implementation.
- Twilio phone verification.
- Stripe payment.
- Selfie upload.
- Credential creation and wallet issuance.
- Additional/second license flow.
