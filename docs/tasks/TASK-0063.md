## Product Goal

Ensure subscription, entitlement, payment, and lapse states are visible enough for MVP ops and safe enough that paid features cannot remain active after lapse.

## Technical Scope

- Define the minimum Supabase dashboard/query views or documented inspection checklist for subscription/payment state.

- Document manual recovery boundaries and audit requirements.

## Codex QA Review

**Date:** 2026-06-04
**Reviewer:** Codex
**Result:** Blocked - needs Claude remediation before this task can pass QA.

### Findings

- **P1 - ops Query 1 references non-existent v4 wallet/pass columns and tables.** `docs/ops/SUBSCRIPTION_MONITORING.md:47-48` and `docs/tasks/TASK-0063-OPS-HARDENING-SPEC.md:46-47` use `wallet_passes.pass_id` and `passes`, but v4 wallet passes use `wallet_passes.credential_id` referencing `credentials.id` (`docs/architecture/V4_MIGRATION_SQL.md:319-333`). Result: the documented ops inspection query will fail.
- **P1 - manual recovery SQL writes invalid subscription status spelling.** `docs/ops/MANUAL_RECOVERY.md:175-177` sets `subscriptions.status = 'cancelled'`, while v4 allows `canceled` (`docs/architecture/V4_MIGRATION_SQL.md:481-486`). Running the documented recovery update will violate the check constraint.
- **P1 - webhook payment records do not match the v4 `payments.action_type` constraint.** `stripe-webhook` writes `subscription_start` and `subscription_renewal`, but v4 allows only `share_token`, `refresh`, `pdf_export`, and `additional_license`. That undermines TASK-0063's claim that payment history is inspectable through `payments` until schema/function behavior is reconciled.
- **P2 - recovery docs query a non-existent `stripe_events.stripe_subscription_id` column.** `docs/ops/MANUAL_RECOVERY.md:157-160` filters `stripe_events` by `stripe_subscription_id`; v4 stores Stripe event data in `payload`, so the query should inspect JSON payload fields or join via `subscriptions`.

- Replace `stripe_events.stripe_subscription_id` references with JSON payload extraction or a documented join path.
- Re-run SQL/runbook QA after remediation.
