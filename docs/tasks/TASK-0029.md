## Product Goal

Persist Stripe subscription/payment state in Supabase and enforce launch-critical entitlement behavior server-side.

## Technical Scope

- Implement Stripe checkout creation or approved subscription start flow.
- Implement Stripe webhook handling with `stripe_events` idempotency.
- Persist `subscriptions` and `payments` state.
- Enforce server-side entitlement checks for paid-tier actions.
- Implement subscription lapse behavior: downgrade to Free behavior while retaining credential/wallet records.
- Write audit/event records for payment and subscription lifecycle events.

## Out of Scope

- Annual plans, coupons, institutional billing, and employer billing.
- Additional license purchase flow unless David separately reopens it.
- Stripe live-mode cutover.

## Acceptance Criteria

- [ ] Standard and Premier monthly subscription states can be persisted.
- [ ] Stripe webhook events are idempotent.
- [ ] Server-side entitlement checks gate launch-critical paid actions.
- [ ] Subscription lapse downgrades account behavior without deleting credential/wallet records.
- [ ] Stripe secrets are not exposed to Lovable.
- [ ] Claude documents tests, files changed, deviations, and risks.
