## Product Goal

Persist wallet pass issuance state in Supabase so nurses, ops, and verifier-facing systems can rely on durable backend state rather than transient provider responses.

## Technical Scope

- Implement or revise backend writes to `wallet_passes`.
- Store provider, status, pass URL or provider reference, issued/updated timestamps, and error state where applicable.
- Link wallet pass records to the credential record.
- Support Apple and Google as separate provider states or document the approved single-row/provider-variant model.
- Record audit/event state for issuance success, pending, failure, and retry.
- Ensure status readers can distinguish:
  - no wallet attempt yet
  - pending
  - issued
  - failed
  - blocked/do_not_issue

## Out of Scope

- Wallet signing contract decisions covered by TASK-0050.
- Nurse dashboard launch-critical actions covered by Phase 5.
- Full wallet pass lifecycle refresh/revocation automation.
- Show QR and PDF export.
