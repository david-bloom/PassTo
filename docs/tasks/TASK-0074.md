### Phase 2 — Schema and RLS

Apply production v4 schema as baseline to demo project, then add:

**`demo_sessions` table:**
- `id` UUID primary key
- `mode` TEXT NOT NULL CHECK (mode IN ('demo', 'uat')) — DEFAULT only, no UPDATE
- `demo_session_id` UUID unique — immutable after insert
- `created_at` TIMESTAMPTZ DEFAULT now()
- `persona` TEXT DEFAULT 'avery_demo'
- RLS: INSERT service-role only; SELECT service-role only; no UPDATE or DELETE

**`demo_personas` table:** Avery Demo fixture with synthetic license record.

**Fail-closed environment guard** applied at Edge Function cold start:
Reject production project URL, production provider credentials, and production
wallet identifiers. Any match returns 503 and logs `demo.environment_guard_tripped`.

**Synthetic `subscriptions` rows** flagged with `demo_session_id` reference,
non-promotable to production.

**Participant phone cleanup:** `demo-cleanup` Edge Function deletes participant
phone field from session. Presenter console action — not a scheduled function.

### Phase 3 — Provider Simulators (Edge Functions)

#### `identity-simulate`

- Input: `demo_session_id`
- Triggered after participant completes the acknowledgment screen
- Validates session mode; rejects production session IDs
- Returns synthetic successful identity result for Avery Demo
- Does not call ID.me; does not accept real ID.me credentials
- Audit event: `demo.identity_simulated`

#### `license-simulate`

- Input: `license_number` (alphanumeric, 2–20 chars), `demo_session_id`
- Accepts any reasonable alphanumeric value; echoes it into synthetic record
- Returns `{ status: "Active - Simulated", name: "Avery Demo", ... }`
- Reserved prefixes (e.g., `FAIL`) trigger deterministic failure for internal
  testing only — not exposed during moderated nurse sessions
- Labels all returned data as synthetic
- Audit event: `demo.license_simulated`

### Phase 4 — Demo Session Management (Edge Functions)

#### `demo-session-create`

- Creates new `demo_session_id` with immutable `mode = demo | uat`
- Resets Avery Demo `onboarding_step` to `identity`
- Clears prior participant phone from session
- Revokes prior active share links for resettable persona
- Preserves prior audit history keyed by `demo_session_id`
- Restores active synthetic license fixture
- Returns session health status: providers, wallet routes, SMS delivery

#### `demo-entitlement-assign`

- Assigns server-controlled Standard tier without Stripe
- Writes synthetic `subscriptions` row: `plan_name = standard`,
  `license_entitlement_count = 1`, `status = active`
- Row flagged non-promotable; no real Stripe customer or subscription ID

**Decision recorded 2026-06-15:** Standard tier as default. Premier features
are post-MVP and should not appear in demo sessions.

#### `demo-cleanup`

- Deletes participant phone/contact field from active demo session
- Called from presenter console after session close
- Audit event: `demo.participant_data_cleared`

#### `demo-selfie-delete`

- Deletes selfie from demo storage at participant request
- Called from presenter console
- Audit event: `demo.selfie_deleted`

### Phase 5 — Selfie-Verifier Backend Contract
(Required before Stage B — production-compatible design)

This is a new backend capability required by both the demo environment and
eventually production. Design must be production-safe from the start.

#### `verifier-selfie` (new Edge Function)

- Input: valid `verification_token`
- Validates token is active and not expired
- Generates a short-TTL Supabase Storage signed URL (60 seconds) for the
  nurse's selfie
- Never returns or logs the raw storage path
- Returns `{ selfie_available: true, url: "<signed-url>" }` or
  `{ selfie_available: false, reason: "not_provided" }`
- No anonymous database reads of storage path
- Audit event: `verifier.selfie_accessed` (token ID and timestamp only —
  not the URL or path)
- Codex security review required before deployment
