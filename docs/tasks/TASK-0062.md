## Product Goal

Resolve the current policy conflict between the payments docs, which price Free-tier paid actions at $1.99, and implemented Phase 5 share-link behavior, which currently allows Free-tier share-link creation without payment.

David decision recorded 2026-06-04:

```text
Free tier can share, generate QR code, refresh, and PDF export for $1.99.
```

This means Free-tier share links, Show QR verifier access, on-demand refresh, and PDF export are available as $1.99 one-time paid actions. Payment-gated action execution must be backend-controlled and may not create tokens, refresh credentials, or generate PDFs until server-confirmed Stripe payment succeeds.

## Technical Scope

- Review PRD, payments flow, subscription/tier docs, TASK-0056, and live `share-link-create` behavior.
- Reconcile Free-tier paid action behavior across share link, Show QR, on-demand refresh, and PDF export.
- Specify or implement backend flows that execute each $1.99 Free-tier paid action only after Stripe payment succeeds.
- Reconcile TASK-0056 share-link behavior with the recorded Free-tier paid-action decision.
- Ensure Lovable does not directly create `verification_tokens`, trigger paid refresh completion, generate PDFs, or mark payments as successful.
- Ensure paid-action state is auditable and idempotent if implemented.
