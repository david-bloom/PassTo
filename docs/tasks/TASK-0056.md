## Product Goal

Allow an eligible nurse to create a controlled one-time share link for verifier access without exposing raw token storage or allowing client-side entitlement bypass.

## Technical Scope

- Implement or revise a backend share-link creation operation.
- Confirm authenticated nurse owns the credential.
- Require credential state and license status to be shareable.
- Enforce server-confirmed entitlement/payment rules.
- Generate a high-entropy raw token.
- Store only a token hash in `verification_tokens`.
- Use token type `share_link`.
- Apply approved TTL: 72 hours or first successful use, whichever comes first.
- Return the raw share URL/token once to Lovable.
- Write audit/event records for creation and rejection paths.

## Out of Scope

- Show QR token creation.
- PDF QR or PDF export.
- Verifier token validation.
- Verifier UI.
- Stripe checkout implementation beyond reading confirmed entitlement/payment state.
- Employer dashboard or verifier accounts.

### Function: `share-link-create`

**verify_jwt:** true | **Method:** POST | **CORS:** `enroll.passtodigital.com`

### Live Schema Confirmed

| Table | Key columns verified |
|---|---|
| `verification_tokens` | id, profile_id, credential_id, token_hash, token_type (default 'share_link'), status (default 'active'), expires_at, used_at, revoked_at, created_at, updated_at |
| `credentials` | id, profile_id, license_id, status, created_at |
| `licenses` | id, profile_id, normalized_status, data_match_passed |
| `profiles` | account_status, subscription_tier, id_verification_status, id_verification_level |
| `subscriptions` | profile_id, status |
| `audit_events` | actor_id, action, resource_type, resource_id, change_after, created_at |
| RLS on `verification_tokens` | SELECT only — no INSERT policy; service-role required for writes |

### Gate Sequence

1. JWT verification — 401 if missing or invalid
2. Profile: `account_status = 'active'`, `id_verification_status = 'verified'`, `id_verification_level = 'IAL2'` — 403 on failure
3. Primary credential: most recent for profile, `status = 'active'` — 403 if not found or not active
4. License linked to credential: `normalized_status = 'Active'`, `data_match_passed = true` — 403 on failure
5. Entitlement: `subscription_tier = 'free'` — allow; any other tier — require active `subscriptions` row — 403 `subscription_not_confirmed` if missing

### Token Generation

- 32 bytes via `crypto.getRandomValues` — 64-char hex raw token
- SHA-256 hash of raw token stored as `token_hash`
- Raw token returned once in response; never stored

### TTL

- `expires_at` = `now() + 72 hours`
- First-use expiry enforced in TASK-0057 (verifier validation function)

### Write Order

1. Audit event (`action: 'create'`, `resource_type: 'verification_token'`) — fail-closed, aborts on insert error
2. `verification_tokens` insert (service-role) — returns 500 if fails; raw token not returned on failure

### Response Payload

```json
{
  "share_url": "https://passtodigital.com/v/{raw_token}",
  "expires_at": "2026-06-05T...",
  "token_type": "share_link"
}
```

`SHARE_LINK_BASE_URL` Supabase secret overrides the default base URL.

### Deviations

- **TASK-0055 not yet executed:** Dashboard UI that surfaces the share-link action has not been built. Backend function is independent and deployable. Lovable will wire up the call when TASK-0055 is approved.
- **Rejection paths do not write audit events:** Gate failure paths return 403 without an audit record. Only successful creation is audited. Codex may recommend rejection auditing.
- **Credential selected by most-recent row:** No `is_primary` flag exists on `credentials`. Function selects `ORDER BY created_at DESC LIMIT 1`. Acceptable for MVP single-license path.

### Risks

- **First-use expiry deferred to TASK-0057:** The `used_at` field and `status = 'used'` enforcement belong to the verifier validation function. Until TASK-0057 is deployed, a share-link token could technically be used multiple times. The dashboard should present the URL as single-use.
- **`SHARE_LINK_BASE_URL` must be set before production use:** Defaults to `https://passtodigital.com/v`. Confirm the verifier base URL before deployment.
