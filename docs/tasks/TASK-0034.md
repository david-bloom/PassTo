## Product Goal

Let the nurse enter license details, run a backend-owned license lookup, persist source data and normalized status, and advance only successful lookup results to data matching.

## Technical Scope

- Implement or wire the dedicated license lookup backend function.
- Accept authenticated nurse license lookup requests from Lovable.
- Call the approved license lookup source from server-side code only.
- Persist lookup attempts and license source data in Supabase.
- Normalize license status according to canonical mappings.
- Return safe frontend responses without raw provider payloads.

## Out of Scope

- Data matching logic.
- Credential issuance.
- Manual review/admin override flow.
- Scheduled refresh or additional license flows.

## Acceptance Criteria

- [ ] Nurse can submit state, license type, and license number or approved required fields.
- [ ] Backend performs the provider call and stores lookup attempt/result.
- [ ] Lookup success advances to data matching.
- [ ] Not found, coverage gap, provider error, and inactive/non-eligible status block issuance.
- [ ] Raw provider payload is not returned to Lovable.
- [ ] Claude documents tests, files changed, deviations, and risks.

## QA Plan

- Test active, inactive, not-found, provider-error, and invalid-input cases.
- Verify `license_lookups` and `licenses` records.
- Verify safe frontend response shape.
