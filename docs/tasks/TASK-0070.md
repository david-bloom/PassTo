## Product Goal

Create a repeatable launch smoke-test script that proves the first production nurse journey works end to end before David approves launch.

## Technical Scope

- Convert PRD Section 7.7 into a step-by-step smoke-test script.
- Cover the full flow: create nurse account, verify identity, verify phone, look up license, pass data match, upload selfie, issue credential, issue wallet pass, view dashboard, create share link, verifier views credential, subscription/payment state behaves correctly, and ops can inspect events.
- Define prerequisites, required test persona/data, environment, evidence capture, pass/fail criteria, and rollback/retry notes.
- Include a dev-only dry run path using TASK-0044 seeded personas before any production-launch exercise.
- Mark any production or live-provider smoke-test step as requiring David approval.

## Out of Scope

- Executing the smoke test.
- Creating production test nurses.
- Running live provider calls without separate approval.
- Fixing failures found during smoke testing.
- Approving production launch.

## Security/RLS Impact

Security Impact: High

The script must include negative checks for direct client bypasses, account-status enforcement, token first-use/expiry behavior, and no secret exposure during evidence capture.

## Design System Impact

Design System Impact: Low
Design Source: `docs/design_system/README.md`

Smoke testing should capture visible launch-blocking UI failures but must not expand design scope.
