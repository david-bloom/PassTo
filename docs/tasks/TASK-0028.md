## Product Goal

Let a verifier open `/v/{token}`, submit minimal verifier information, accept Terms, and view safe credential status through a tokenized flow without a PassTo account.

## Technical Scope

- Implement token validation backend operation.
- Implement `/v/{token}` Lovable verifier route.
- Collect minimal verifier information: name and email.
- Require Terms of Use acceptance.
- Create `verifiers` and `verification_events` records.
- Mark token used after successful verification.
- Return safe credential/license/current-as-of display data only.

## Out of Scope

- Verifier account creation.
- Employer dashboard.
- Show QR and PDF flows.

## Acceptance Criteria

- [ ] Valid unused token lets verifier submit name/email and Terms acceptance.
- [ ] Expired, used, revoked, missing, or invalid token shows safe failure behavior.
- [ ] Successful verification creates verifier and verification event records.
- [ ] Token is marked used after successful verification.
- [ ] Verifier cannot browse records outside the tokenized flow.
- [ ] Claude documents tests, files changed, deviations, and risks.

## QA Plan

- Test valid, expired, used, revoked, malformed, and unknown token paths.
- Verify event writes and token status transition.
- Verify returned credential display excludes private nurse/internal data.
