## Product Goal

Capture and store the nurse selfie as a launch-critical trust gate before credential creation.

## Technical Scope

- Create or verify protected Supabase Storage bucket/policies for selfies.
- Implement Lovable selfie capture/upload flow.
- Associate uploaded selfie asset with the authenticated nurse/profile.
- Prevent public access to selfie assets.
- Persist capture success/failure state needed for credential creation.
- Route failed or abandoned selfie capture to retry/support behavior.

## Out of Scope

- ID.me-owned selfie storage.
- Face matching or biometric comparison unless separately approved.
- Credential creation and wallet issuance.

## Acceptance Criteria

- [ ] Nurse can capture/upload selfie after data match passes.
- [ ] Selfie asset is stored in protected Supabase Storage.
- [ ] Nurse cannot access another nurse's selfie asset.
- [ ] Credential creation remains blocked until selfie capture succeeds.
- [ ] Failure/abandonment path is clear and auditable.
- [ ] Claude documents tests, files changed, deviations, and risks.

## QA Plan

- Test upload success, upload failure, unauthorized access, and retry behavior.
- Verify Storage bucket access policies.
- Verify selfie state is associated with the correct nurse.
