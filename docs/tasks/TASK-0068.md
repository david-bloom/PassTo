## Product Goal

Create a complete MVP QA plan that covers PassTo's launch-critical user journey, integration failure states, account enforcement, security boundaries, and ops visibility before production launch.

## Technical Scope

- Convert PRD Section 7.4 and 7.5 into an executable QA plan.
- Define test cases for account creation, login/session persistence, ID.me, phone verification, license lookup, data match, selfie upload, credential creation, wallet issuance, dashboard, share link, verifier token flow, verifier Terms acceptance, Stripe, subscription lapse, account-status enforcement, seeded personas, RLS, failure logging, and ops visibility.
- Identify manual QA steps, expected results, required personas, test data, and evidence to capture.
- Separate read-only QA from QA that requires live writes, provider calls, deployment, payment actions, or David approval.
- Produce a reusable QA matrix suitable for Codex review and Claude execution handoff.

## Out of Scope

- Executing the full QA plan.
- Creating or changing seed data outside TASK-0044.
- Fixing QA findings.
- Running live production provider calls unless separately approved.
- Approving production launch.
