## Product Goal

Give the nurse a trustworthy final enrollment status surface that shows credential creation and wallet readiness without overstating completion when issuance is pending or failed.

## Technical Scope

- Implement or revise `/success` / PassReady status behavior.
- Read credential and wallet status from backend-owned status endpoint(s).
- Show states for:
  - credential created, wallet issued
  - credential created, wallet pending
  - credential created, wallet failed
  - credential blocked or not ready
  - delayed provider response
- Provide safe wallet links when available.
- Provide retry/support messaging for wallet issuance failure.
- Preserve `/pass-ready` redirect/alias/removal decision once David chooses final disposition.
- Keep add-another-license, share-link, Show QR, and PDF export out of this Phase 4 surface unless separately approved.

## Out of Scope

- Nurse dashboard implementation beyond routing after success.
- Share-link creation.
- Show QR.
- PDF export.
- Wallet provider signing implementation.
- Credential issuance gate implementation.

## Security/RLS Impact

Security Impact: Medium

The page reads sensitive credential/wallet state. It must use authenticated, user-scoped backend reads and safe response payloads.

## Design System Impact

Design System Impact: Medium  
Design Source: `/docs/design_system/README.md`

Lovable should use existing PassTo design conventions, clear loading/error states, and avoid marketing-style layout. The page is an operational status surface.

## Acceptance Criteria

- [ ] `/success` reads credential/wallet status from backend-owned data.
- [ ] The UI distinguishes issued, pending, failed, and blocked states.
- [ ] Wallet URLs/actions appear only when backend says they are available.
- [ ] A failed wallet issuance does not hide the created credential or pretend the pass is ready.
- [ ] `/pass-ready` disposition is implemented only after David approval.
- [ ] No client-side trust decision marks credential or wallet issuance complete.
- [ ] Claude/Lovable documents files changed, tests run, deviations, and risks.
