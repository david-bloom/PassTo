### ID.me OAuth Flow

```
1. Nurse at /id-verification clicks "Verify with ID.me"
2. IdVerification.tsx (launchIdmeAuth):
   a. Generates random state:        crypto.randomUUID()
   b. Generates PKCE code_verifier:  64-char random base64url string
   c. Computes code_challenge:       BASE64URL(SHA-256(ASCII(code_verifier)))
   d. Stores in sessionStorage:      'idme_state' = state
                                     'idme_code_verifier' = code_verifier
   e. Builds authorization URL with all params (see below)
3. Browser redirects to:
     https://<VITE_IDME_BASE_URL>/oauth/authorize
       ?client_id=<VITE_IDME_CLIENT_ID>
       &redirect_uri=<VITE_IDME_REDIRECT_URI>
       &response_type=code
       &scope=tefca
       &state=<state>
       &code_challenge=<code_challenge>
       &code_challenge_method=S256
4. Nurse completes ID.me verification
5. ID.me redirects to:
     https://enroll.passtodigital.com/auth/idme/callback?code=<auth_code>&state=<state>
     (On nurse cancellation or error: ?error=access_denied&state=<state>)
6. IdmeCallback.tsx:
   a. Reads code, state, error params from URL
   b. If error param is present → show cancellation/error state; offer restart
   c. Reads stored state from sessionStorage('idme_state')
   d. Validates returned state === stored state
      - Mismatch or missing → security error; no Edge Function call; offer restart
   e. Reads code_verifier from sessionStorage('idme_code_verifier')
   f. Clears 'idme_state' and 'idme_code_verifier' from sessionStorage
   g. Verifies active Supabase session → if none, navigate to /login
   h. Calls:
        supabase.functions.invoke("idme-exchange", { body: { code, code_verifier } })
7. idme-exchange Edge Function:
   a. Verifies JWT → auth_user_id
   b. Exchanges code + code_verifier with ID.me token endpoint (server-side)
   c. Validates IAL2 + TEFCA scope
   d. Updates profiles (service-role)
   e. Writes audit_events
   f. Returns { verified: true } or { verified: false, error: "<key>" }
8. IdmeCallback.tsx reads result:
     - verified: true  → navigate to /verify-phone
     - verified: false → show error state with error key
```

## Edge Function: `idme-exchange`

### Purpose

Server-side ID.me OAuth token exchange. Replaces the previous browser-to-Make `fetch()` pattern (CF-3). The browser never handles the client secret.

### Invocation

Called from `IdmeCallback.tsx` via `supabase.functions.invoke("idme-exchange", { body: { code } })`. The user JWT is automatically included in the Authorization header by the Supabase client.

### Input

| Field | Type | Source | Required |
|---|---|---|---|
| `code` | string | ID.me OAuth callback `?code=` param | Yes |
| `code_verifier` | string | PKCE verifier from `sessionStorage('idme_code_verifier')` | Yes |
| JWT | string | Auto-sent in Authorization header by Supabase client | Yes |

### Process

```
1. Validate input:
   - code must be present and non-empty string
   - code_verifier must be present and non-empty string (43–128 chars, unreserved chars only)
   - if either missing → return 400 { error: 'missing_code' } or { error: 'missing_code_verifier' }
2. Verify JWT:
     const { data: { user }, error } = await supabaseAdmin.auth.getUser(jwt)
     if (error || !user) → return 401
3. Look up profile:
     const { data: profile } = await supabaseAdmin
       .from('profiles')
       .select('id, id_verification_status')
       .eq('auth_user_id', user.id)
       .maybeSingle()
     if (!profile) → return 404 { error: 'profile_not_found' }
4. Exchange code with ID.me token endpoint (PKCE included):
     POST https://<IDME_TOKEN_URL>
       Content-Type: application/x-www-form-urlencoded
       grant_type=authorization_code
       &code=<code>
       &client_id=<IDME_CLIENT_ID>
       &client_secret=<IDME_CLIENT_SECRET>
       &redirect_uri=<IDME_REDIRECT_URI>
       &code_verifier=<code_verifier>
     if error → return 200 { verified: false, error: 'exchange_failed' }
5. Fetch user attributes from ID.me:
     GET https://<IDME_ATTRIBUTES_URL>
       Authorization: Bearer <access_token>
6. Validate IAL level:
     if attributes.ial < 2 or scope is not TEFCA:
       → update profiles: id_verification_status='failed', id_verification_level='IAL1'
       → write audit: identity.verification_failed
       → return 200 { verified: false, error: 'ial1_only' }
7. Extract id_me_subject (ID.me unique user identifier — 'uuid' or 'sub' field)
8. Check for duplicate subject:
     if another profile already has this id_me_subject:
       → write audit: identity.verification_failed
       → return 200 { verified: false, error: 'identity_already_registered' }
9. Update profile (service-role):
     id_verification_status = 'verified'
     id_verification_level  = 'IAL2'
     id_me_subject          = <subject>
     onboarding_step        = 'phone'
10. Write audit event:
     action:        'identity.verification_completed'
     actor_id:      profile.id
     resource_type: 'profile'
     resource_id:   profile.id
     change_after:  { id_verification_status: 'verified', id_verification_level: 'IAL2', onboarding_step: 'phone' }
11. Return 200 { verified: true }
```

### Idempotency

If a nurse calls `idme-exchange` after already being verified (`id_verification_status = 'verified'`):
- The ID.me auth code is single-use and will be rejected by ID.me's token endpoint
- The Edge Function should return `{ verified: false, error: 'exchange_failed' }` in this case
- The nurse is already verified — IdmeCallback.tsx should check profile state as a fallback (see Section below)

### Failure Cases and Return Values

| Condition | HTTP status | Body |
|---|---|---|
| `code` missing or empty | 400 | `{ error: 'missing_code' }` |
| `code_verifier` missing or empty | 400 | `{ error: 'missing_code_verifier' }` |
| JWT invalid or missing | 401 | `{ error: 'unauthenticated' }` |
| Profile not found | 404 | `{ error: 'profile_not_found' }` |
| ID.me code exchange error (incl. PKCE mismatch) | 200 | `{ verified: false, error: 'exchange_failed' }` |
| IAL1 returned (not IAL2) | 200 | `{ verified: false, error: 'ial1_only' }` |
| Subject already registered | 200 | `{ verified: false, error: 'identity_already_registered' }` |
| Unexpected/provider error | 200 | `{ verified: false, error: 'provider_error' }` |
| Success | 200 | `{ verified: true }` |

**Changes:**

```tsx
// PKCE helper functions (module-level, not exported):
const generateCodeVerifier = (): string => {
  const array = new Uint8Array(48);
  crypto.getRandomValues(array);
  return btoa(String.fromCharCode(...array))
    .replace(/\+/g, '-').replace(/\//g, '_').replace(/=/g, '');
};

const generateCodeChallenge = async (verifier: string): Promise<string> => {
  const encoder = new TextEncoder();
  const data = encoder.encode(verifier);
  const digest = await crypto.subtle.digest('SHA-256', data);
  return btoa(String.fromCharCode(...new Uint8Array(digest)))
    .replace(/\+/g, '-').replace(/\//g, '_').replace(/=/g, '');
};

// Replace the hardcoded IDME_AUTHORIZE_URL constant and handleVerify no-op:
const handleVerify = async () => {
  const base = import.meta.env.VITE_IDME_BASE_URL;
  const clientId = import.meta.env.VITE_IDME_CLIENT_ID;
  const redirectUri = import.meta.env.VITE_IDME_REDIRECT_URI;

  const state = crypto.randomUUID();
  const codeVerifier = generateCodeVerifier();
  const codeChallenge = await generateCodeChallenge(codeVerifier);

  sessionStorage.setItem('idme_state', state);
  sessionStorage.setItem('idme_code_verifier', codeVerifier);

  const params = new URLSearchParams({
    client_id: clientId,
    redirect_uri: redirectUri,
    response_type: 'code',
    scope: 'tefca',
    state,
    code_challenge: codeChallenge,
    code_challenge_method: 'S256',
  });

  window.location.href = `${base}/oauth/authorize?${params.toString()}`;
};

// Button: remove disabled, restore original label, add async onClick
<Button className="w-full" size="lg" onClick={handleVerify}>
  Verify with ID.me
  <ArrowRight className="ml-2 h-4 w-4" />
</Button>
```

### `IdmeCallback.tsx`

Replace the stub useEffect with a real implementation that calls `idme-exchange`.

**Changes:**

```tsx
useEffect(() => {
  if (ranRef.current) return;
  ranRef.current = true;

  const code = searchParams.get("code");
  const returnedState = searchParams.get("state");
  const idmeError = searchParams.get("error");

  // ID.me returned error (e.g. access_denied from nurse cancellation)
  if (idmeError) {
    setErrorKey(idmeError === "access_denied" ? "access_denied" : null);
    setStatus("error");
    sessionStorage.removeItem('idme_state');
    sessionStorage.removeItem('idme_code_verifier');
    return;
  }

  if (!code) {
    setErrorKey(null);
    setStatus("error");
    return;
  }

  // CSRF state validation
  const storedState = sessionStorage.getItem('idme_state');
  if (!returnedState || returnedState !== storedState) {
    setErrorKey("state_mismatch");
    setStatus("error");
    sessionStorage.removeItem('idme_state');
    sessionStorage.removeItem('idme_code_verifier');
    return;
  }

  const codeVerifier = sessionStorage.getItem('idme_code_verifier');

  // Clear PKCE/state material immediately — single use
  sessionStorage.removeItem('idme_state');
  sessionStorage.removeItem('idme_code_verifier');

  if (!codeVerifier) {
    setErrorKey(null);
    setStatus("error");
    return;
  }

  (async () => {
    // Verify active session before calling Edge Function
    const { data: { session } } = await supabase.auth.getSession();
    if (!session) {
      navigate("/login");
      return;
    }

    const timeoutId = window.setTimeout(() => {
      setStatus("timeout");
    }, 30000);

    try {
      const { data, error } = await supabase.functions.invoke("idme-exchange", {
        body: { code, code_verifier: codeVerifier },
      });

      window.clearTimeout(timeoutId);

      if (error) {
        setErrorKey(null);
        setStatus("error");
        return;
      }

      if (data?.verified === true) {
        navigate("/verify-phone");
        return;
      }

      setErrorKey(typeof data?.error === "string" ? data.error : null);
      setStatus("error");
    } catch (err) {
      window.clearTimeout(timeoutId);
      setErrorKey(null);
      setStatus("error");
    }
  })();
}, [searchParams, navigate]);
```

## Failure State Routing

| Failure | Edge Function returns | Lovable behavior |
|---|---|---|
| Nurse cancels at ID.me (`?error=access_denied`) | (pre-flight check) | Show "Verification cancelled" error; offer restart |
| State mismatch or missing | (pre-flight check) | Show security error; offer restart; no Edge Function call |
| Missing code_verifier in sessionStorage | (pre-flight check) | Show error state; offer restart |
| IAL1 returned | `{ verified: false, error: 'ial1_only' }` | Show "Verification incomplete" error; offer restart |
| Identity already registered | `{ verified: false, error: 'identity_already_registered' }` | Show "Identity already registered" error; offer login link |
| Code exchange error (incl. PKCE mismatch at ID.me) | `{ verified: false, error: 'exchange_failed' }` | Show "Verification could not complete" error; offer restart |
| Provider error | `{ verified: false, error: 'provider_error' }` | Show "ID.me temporarily unavailable" error; offer restart |
| Timeout (30s) | (no response) | Show "Taking longer than expected" timeout state; offer restart |
| No session at callback | (pre-flight check) | Redirect to `/login` before calling Edge Function |
| Missing code in URL | (pre-flight check) | Immediately show error state; offer restart |

The `restart()` function navigates back to ID.me authorization. It does not retry the same code.

## Security Boundaries

### What Lovable may do in Phase 3.1

| Action | Mechanism |
|---|---|
| Generate state and PKCE code_verifier/challenge | `crypto.randomUUID()` and `crypto.subtle.digest()` — browser-native, no secret required |
| Store state and code_verifier | `sessionStorage` — tab-scoped, cleared immediately after use, never sent to server |
| Redirect nurse to ID.me authorization URL | `window.location.href` with all params including state, code_challenge, code_challenge_method — uses browser-safe env vars only |
| Receive OAuth callback code and state from URL | `searchParams.get("code")` and `searchParams.get("state")` — code passed to Edge Function; state validated locally first |
| Call idme-exchange Edge Function | `supabase.functions.invoke("idme-exchange", { body: { code, code_verifier } })` — user JWT auto-attached |
| Display result and route | Read `data.verified` and `data.error` from Edge Function response |

### What Lovable must never do

| Forbidden action | Reason |
|---|---|
| Exchange the OAuth code with ID.me directly from the browser | CF-3 violation; exposes client_secret or requires browser to hold it |
| Skip state validation before calling idme-exchange | CSRF protection; state mismatch must block all downstream calls |
| Reuse a code_verifier across sessions or page reloads | PKCE material is single-use; always generate fresh state/verifier per authorization attempt |
| Store `IDME_CLIENT_SECRET` in any Vite env var | Must be Edge Function secret only |
| Write `profiles.id_verification_status` directly | Trust-gate field; service-role only; no authenticated UPDATE RLS policy |
| Write `profiles.onboarding_step` directly | Trust-gate field; service-role only |
| Trust the ID.me verification result based on client-side callback params alone | Server-side exchange required |
