## Product Goal

Create the PassTo credential record only after all launch-critical trust gates pass: ID.me IAL2, phone verification, license lookup, data match, and selfie capture.

## Technical Scope

- Implement backend credential creation operation.
- Enforce all prerequisite gates server-side.
- Set credential status, license status, current-as-of timestamp, and wallet pass treatment.
- Write audit/event records for credential creation success/failure.
- Prevent Lovable from directly creating credentials.

## Out of Scope

- Wallet pass signing/issuance.
- Share-link verifier flow.
- Stripe checkout/webhooks.

## Acceptance Criteria

- [ ] Backend refuses credential creation unless all trust gates pass.
- [ ] Credential row is created with correct profile/license relationship.
- [ ] Credential status and current-as-of fields are populated.
- [ ] Failure path writes event/audit state and does not partially issue wallet pass.
- [ ] Lovable cannot bypass backend gate by writing directly.
- [ ] Claude documents tests, files changed, deviations, and risks.
