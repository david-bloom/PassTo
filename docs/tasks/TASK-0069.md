## Product Goal

Create a production configuration checklist that prevents launch from depending on undocumented environment variables, redirects, secrets, domains, provider mode choices, or unsafe dev tooling.

## Technical Scope

- Convert PRD Section 7.6 into a production configuration checklist.
- Document required production configuration categories for Supabase, Lovable domains, Vercel wallet routes, Stripe, ID.me, Twilio, RapidAPI/Propelus, Postmark, wallet signing, auth redirects, CORS, and dev/test seed guards.
- Include Supabase Auth password-security controls for the real project `wvzjfxacykgsaffskgtr`, including minimum password policy and leaked-password protection through the HIBP check.
- For each category, identify owner, environment, evidence source, verification method, and approval status.
- Separate test-mode, sandbox-mode, and live-mode requirements.
- Include explicit checks that dev/test seed tooling cannot run against production.
