## Product Goal

Make sure all launch-affecting open decisions are visible, routed, and either resolved, explicitly deferred, or blocked before production launch.

## Technical Scope

- Review PRD Section 7.8 and current GitHub source-of-truth docs for open decisions.
- Reconcile `docs/decisions/DECISIONS_LOG.md`, `docs/activity_log/DECISIONS_LOG.md`, `docs/activity_log/APPROVALS_LOG.md`, and relevant task docs.
- Create or update a launch decision register that lists each open decision, owner, current state, dependent tasks, launch impact, next action, and required approval.
- Cover at minimum: v4 migration SQL approval, production vs dev Supabase project usage, wallet signing implementation details, Twilio production readiness/fallback, Terms/verifier disclosure language, and exact launch smoke-test data/test nurse.
- Identify stale, duplicated, or superseded decision entries without deleting source-of-truth history.

## Out of Scope

- Making David decisions on David's behalf.
- Closing decisions without documented approval.
- Editing legal/Terms language beyond routing the decision.
- Changing provider configuration, migrations, production data, code, or deployment state.
- Approving production launch.
