## Security/RLS Impact

Security Impact: High

## Design System Impact

Design System Impact: Medium  
Design Source: `/docs/design_system/README.md`

Codex should review whether `/success` status states are clear, non-misleading, and consistent with the product/status UI.

## Acceptance Criteria

- [ ] Credential creation refuses every missing/failed trust gate.
- [ ] Successful credential creation writes correct canonical state.
- [ ] Wallet signing contract keeps secrets backend-only.
- [ ] Wallet provider state is durable in `wallet_passes`.
- [ ] Wallet failure/pending states are visible and do not misrepresent status.
- [ ] `/success` direct navigation cannot skip gates.
- [ ] Audit/event records exist for success and failure.
- [ ] RLS/direct client access cannot mutate credential/wallet trust state.
- [ ] Codex records findings and next required action in GitHub.
