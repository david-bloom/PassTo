## Product Goal

Add the hard credential issuance gate that conservatively verifies the ID.me identity belongs to the license record returned by license lookup.

## Technical Scope

- Implement deterministic backend data matching using approved matching rules.
- Compare verified ID.me name and DOB against license holder data.
- Support conservative name-only matching only when license DOB is unavailable.
- Persist match result and metadata.
- Block credential issuance on mismatch, low assurance, missing ID.me data, or missing license holder data.
- Clear temporary identity fields only after durable match/audit evidence exists.

## Out of Scope

- Manual review override.
- Broad fuzzy matching or unapproved nickname matching.
- Credential creation or wallet issuance.

## Acceptance Criteria

- [ ] Match pass requires verified ID.me status and IAL2.
- [ ] DOB mismatch fails when license DOB is available.
- [ ] Name mismatch fails.
- [ ] Name-only pass records `dob_match_mode = unavailable_name_only` or equivalent metadata.
- [ ] Failed match blocks credential issuance and writes audit/event state.
- [ ] Claude documents tests, files changed, deviations, and risks.

## QA Plan

- Unit or fixture tests for exact match, middle-name variance, DOB mismatch, first-name mismatch, last-name mismatch, missing ID.me, and missing license holder data.
- Verify pass/fail state persistence.
- Verify temporary identity fields are not exposed to Lovable.
