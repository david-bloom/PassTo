## Product Goal

Allow an entitled nurse to create a controlled, short-lived share link for verifier access.

## Technical Scope

- Implement backend share-link token creation.
- Confirm nurse ownership of credential.
- Enforce entitlement checks server-side.
- Store only token hash in Supabase.
- Return raw token/link once to Lovable.
- Enforce approved TTL and status values.
- Write token creation audit/event records.

## Out of Scope

- Show QR.
- PDF QR.
- Verifier view implementation beyond returning the link.

## Acceptance Criteria

- [ ] Entitled nurse can create a share-link token for their own credential.
- [ ] Unentitled nurse or wrong-owner request is rejected server-side.
- [ ] Raw token is never stored.
- [ ] Token hash, expiry, status, and credential/profile relationships are persisted.
- [ ] Creation event/audit records are written.
- [ ] Claude documents tests, files changed, deviations, and risks.

## QA Plan

- Test entitled, unentitled, wrong-owner, expired credential, and revoked credential cases.
- Verify raw token absence from database.
- Verify TTL and audit/event records.
