## Product Goal

Make the remaining onboarding backend state and routing match the ID.me-first flow:

```text
/phone-check
→ /account-select
→ /payment if paid
→ /upload-selfie
→ /success
```

## Technical Scope

- Update onboarding state/routing so phone verification follows successful license lookup and ID.me/license binding.
- Ensure `/phone-check` sends and verifies Twilio OTP through backend-only trusted operations.
- Ensure plan selection happens after PassTo has demonstrated verified identity + matched license value.
- Ensure Free plan skips payment.
- Ensure Standard/Premier route to Stripe payment and require webhook/server confirmation before paid entitlement is active.
- Ensure `/upload-selfie` follows plan/payment handling according to final selfie requirement.
- Add or alias `/success` as the final success route, replacing or superseding older `/pass-ready` in new UX work.
- Ensure `/success` can show wallet readiness and support add-another-license only when the selected plan allows it.
- Update backend route guards so direct URL navigation cannot skip required gates.

## Out of Scope

- ID.me-first attempt implementation.
- License lookup/provider integration.
- Credential creation implementation, except route gating needed to reach success safely.
- Wallet signing/certificate setup.
- Live Stripe product creation unless separately approved.
- Live Twilio production enablement before A2P approval.
